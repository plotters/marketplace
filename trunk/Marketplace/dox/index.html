What was a separate MPN application now run as a timer task inside Marketplace

How does it work ?

MarketplaceNotifier (MPN) is a WebObjects application, but one which runs in an unusual manner. All the work of MPN is done in the Application class, which then terminates. This means that no Sessions are ever created and so this isn't really a web application since it never presents any way for users to see a web page. This is not a technical requirement, it's a convenience which lets us use EOModels and Frameworks that the real Marketplace application uses.

Files/Directories required for MarketplaceNotifier (MPN)

MPN uses two local frameworks:
  • OracleDriver.framework -- this framework is a convenient vehicle for the Oracle JDBC class library. Presently it is carrying the ojdbc14.jar library.
  • JakartaCommons.framework -- this framework carries a suite of Jakarta Commons jar files. MPN uses the commons-lang library for some methods to perform time arithmetic.
MPN uses three files/bundles in Resources:
  • ITCSUser.eomodeld -- trivially relates uniqname to an index that the Marketplace database uses.
  • MarketPlace.eomodeld -- this is the Marketplace database.
  • Properties -- allows the application to be conditioned; in particular for testing the property mpn.testApplication should be set to TRUE and the property mpn.testUser set to the uniqname of the person who should get all the notification messages.

How to Run on Mac OS X

To run this application from XCode just run the application.

To run this application at the command line, just execute it from its deployed location:

    /Library/WebObjects/Applications/MarketplaceNotifier.woa/MarketplaceNotifier

How to Run on Linux

To run this application at the command line, just execute it from its deployed location: 
    /opt/Apple/Local/Library/WebObjects/Applications/MPN.woa/MPN
Remember that, in a Linux deployment, the frameworks are embedded in the application bundle at 
    ..../MPN.woa/MPN/Contents/Frameworks/
